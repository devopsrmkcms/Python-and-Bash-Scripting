# 🚀 ✅ Top Subprocess One-Liners (DevOps Focus)

## 🔹 Basic Execution

```python
import subprocess; subprocess.run(["ls", "-l"])
```

👉 Run a command (safe & modern)

***

## 🔹 Capture Output ⭐

```python
import subprocess; result = subprocess.run(["ls"], capture_output=True, text=True); print(result.stdout)
```

👉 ✅ **MOST USED in DevOps**  
👉 Get command output inside Python

***

## 🔹 Check Command Success ⭐

```python
import subprocess; subprocess.run(["systemctl", "status", "nginx"], check=True)
```

👉 Raises error if command fails

***

## 🔹 Get Return Code ⭐

```python
import subprocess; result = subprocess.run(["ls"]); print(result.returncode)
```

👉 0 = success, non-zero = failure

***

## 🔹 Run Shell Command

```python
import subprocess; subprocess.run("df -h", shell=True)
```

👉 Run command like in terminal

⚠️ Avoid in production (security risk)

***

## 🔹 Capture stdout + stderr ⭐

```python
import subprocess; r = subprocess.run(["ls","/wrong"], capture_output=True, text=True); print(r.stderr)
```

👉 ✅ Debugging failure

***

## 🔹 Use PIPE manually

```python
import subprocess; p = subprocess.Popen(["ls"], stdout=subprocess.PIPE); print(p.stdout.read())
```

👉 Low-level control

***

## 🔹 Run with Timeout ⭐

```python
import subprocess; subprocess.run(["sleep","10"], timeout=2)
```

👉 ✅ Critical for monitoring scripts

***

## 🔹 Send Input

```python
import subprocess; subprocess.run(["grep","test"], input="test\n", text=True)
```

👉 Pass data to command

***

## 🔹 Check if process running ⭐

```python
import subprocess; print("nginx" in subprocess.run(["ps","-ef"], capture_output=True, text=True).stdout)
```

👉 DevOps monitoring use

***

## 🔹 Run and Get Output Quickly ⭐

```python
import subprocess; output = subprocess.getoutput("hostname"); print(output)
```

👉 ✅ Shortcut (very popular)

***

## 🔹 Run Multiple Commands (chain)

```python
import subprocess; subprocess.run("cd /tmp && ls", shell=True)
```

***

## 🔹 Background Process

```python
import subprocess; subprocess.Popen(["sleep", "10"])
```

👉 Run in background

***

## 🔹 Redirect Output to File

```python
import subprocess; subprocess.run(["ls"], stdout=open("file.txt","w"))
```

***

## 🔹 Run Command with Environment

```python
import subprocess, os; subprocess.run(["printenv"], env=os.environ)
```

***

# 🎯 ✅ MOST IMPORTANT (DevOps Use)

subprocess.run(..., capture_output=True, text=True)

✅ Used in:

subprocess.run(..., check=True)

# 💡 ✅ Real DevOps Examples

***

## 🔹 Example 1: Check service status

```python
import subprocess
r = subprocess.run(["systemctl","is-active","nginx"], capture_output=True, text=True)
print(r.stdout.strip())
```

***

## 🔹 Example 2: Disk usage alert

```python
import subprocess
disk = subprocess.getoutput("df -h | grep /")
print(disk)
```

***

## 🔹 Example 3: Kubernetes check

```python
import subprocess
pods = subprocess.getoutput("kubectl get pods")
print(pods)
```

***

## 🔹 Example 4: Nagios Style Check

```python
import subprocess, sys

r = subprocess.run(["systemctl","is-active","nginx"], capture_output=True, text=True)

if "active" in r.stdout:
    print("OK - nginx running")
    sys.exit(0)
else:
    print("CRITICAL - nginx down")
    sys.exit(2)
```

***

# ⚠️ ✅ Important Best Practices

***

## ❌ Avoid shell=True (unless needed)

```python
subprocess.run("rm -rf /", shell=True)  # dangerous
```

***

## ✅ Prefer list format

```python
["ls", "-l"]
```

***

## ✅ Always handle errors

```python
try:
    subprocess.run(..., check=True)
except:
    print("Failed")
```
